# Статус дипломного проєкту — IT WMS (Підсистема обліку та підтримання запасів)

> Аналіз виконано на основі: Курсова робота (ПРІНС), Звіт з виробничої практики (ПрАТ НТЦ «Інформаційні системи»), поточного коду проєкту.
> Дата: 2026-04-23

---

## Загальний опис системи

**Об'єкт автоматизації:** відділ системного адміністрування та обчислювальної техніки ПрАТ НТЦ «Інформаційні системи».  
**Предмет:** підсистема обліку та підтримання запасів запасних частин (МтаК).  
**Стек:** React 19 + TypeScript + Vite + Tailwind CSS v4 / FastAPI + MongoDB.

## ✅ Що вже виконано

### Аутентифікація та авторизація
- [x] Сторінка входу (Login) — JWT-аутентифікація, ролевий редирект
- [x] Сторінка реєстрації (Register) — вибір ролі при реєстрації
- [x] Захист маршрутів — `DashboardLayout` з перевіркою ролі та токена
- [x] Збереження сесії в `localStorage` (JWT + user info)
- [x] Ролева модель: **Завідувач складу (manager)** / **Працівник складу (worker)**

### Функціонал Працівника складу
- [x] **Огляд запасів (WorkerDashboard)** — таблиця всіх МтаК зі статусами (`available`, `issued`, `written_off`, `damaged`)
- [x] **Оприходування (Receiving)** — прийом існуючих позицій і реєстрація нових з усіма атрибутами (назва, SKU, категорія, ціна, термін придатності)
- [x] **Видача (Issuing)** — видача матеріалів з фіксацією отримувача, перевірка наявності, документ типу `issuing`
- [x] **Інвентаризація (InventoryCheck)** — звірка фактичних залишків із системними, формування акту розбіжностей `discrepancy_act`
- [x] **Повернення (Returns)** — повернення виданих МтаК на склад, відновлення статусу `available`
- [x] **Списання (WriteOffs)** — створення акту списання `act_writeoff` зі статусом `pending` (очікує затвердження менеджером), фіксація причини

### Функціонал Завідувача складу
- [x] **Аналітична панель (ManagerDashboard)** — 4 KPI-картки, 3 типи графіків (бар, лінія, кругова), фільтри за periodом (день/місяць/квартал), зведена таблиця за категоріями
- [x] **Управління постачальниками (Suppliers)** — перегляд, пошук, додавання, редагування, видалення; поля: назва, ЄДРПОУ, адреса, IBAN, контактна особа, email, Telegram, телефон
- [x] **Закупівлі (Procurement)** — перегляд замовлень зі статусами (Planned/Ordered/Received), формування замовлень для позицій з низьким залишком
- [x] **Акти та накладні (Acts)** — список документів із фільтрацією по типу, завантаження PDF
- [x] **Затвердження (Approvals)** — перегляд актів списання зі статусом `pending`, затвердження з електронним підписом або відхилення

### Управління документами
- [x] Документи типів: `invoice`, `act_writeoff`, `discrepancy_act`, `issuing`
- [x] Workflow: `pending` → `approved` (через затвердження менеджером)
- [x] Архів документів у кожному розділі (пошук по ID)

### Бекенд (FastAPI + MongoDB)
- [x] `POST /auth/register`, `POST /auth/login`, `GET /auth/me`
- [x] CRUD `/suppliers` (захист для manager)
- [x] CRUD `/items` (фільтри за статусом і категорією; delete — тільки manager)
- [x] CRUD `/persons` (manager only)
- [x] `GET/POST /transactions` (типи: `in`, `out`, `write_off`)
- [x] `GET/POST/PUT /documents`, `PATCH /documents/{id}/approve`
- [x] `GET /stats` — агрегована статистика для дашборду
- [x] MongoDB колекції: `users`, `suppliers`, `items`, `persons`, `transactions`, `documents`

### Інфраструктура та UX
- [x] Двомовний інтерфейс — **Українська / English** (254 ключі в кожній мові)
- [x] Генерація PDF — накладні, зведені звіти, акти розбіжностей (`jsPDF` + `autoTable`)
- [x] Централізований API-клієнт (`src/utils/api.ts`) з авто-підстановкою JWT
- [x] React Context для стану: `AuthContext`, `ItemsContext`, `SupplierContext`, `DocumentsContext`, `LanguageContext`
- [x] Адаптивний дизайн (Tailwind CSS v4), тематизація за роллю
- [x] Анімації, статусні бейджи, порожні стани, індикатори завантаження
